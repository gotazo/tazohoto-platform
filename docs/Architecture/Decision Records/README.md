# Architecture Decision Records (ADR)

## Purpose

Architecture Decision Records (ADRs) capture the significant technical decisions made during the development of the Tazohoto Platform.

Each ADR explains:

- The context
- The decision
- The reasoning
- The consequences

ADRs provide long-term architectural history for the platform.

## Naming Convention

```
ADR-0001-Decision-Name.md
```

Example:

```
ADR-0001-Monorepo.md
```

## Lifecycle

- Proposed
- Accepted
- Superseded
- Deprecated

New decisions create new ADRs.

Existing ADRs should not be rewritten.
